## Rules

### Read docs first

Read `documentation/` before editing any markdown files to follow writing standards.

### Follow existing patterns

Match the style and structure of existing documents in this repository.

### Keep it simple

Document only what is necessary. Avoid duplication and over-explanation.

### Priority labels

Use P1/P2/P3 priority labels in section headings as defined in `documentation/document_structure.md`.
